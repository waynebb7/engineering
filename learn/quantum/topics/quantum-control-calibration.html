1) Why calibration matters

Quantum hardware drifts: qubit frequencies shift with temperature, flux crosstalk changes coupling strengths, and amplifier gain varies. Without calibration, gate fidelities degrade and algorithm results become unreliable.

  • Typical recalibration cycle: every few hours to daily for superconducting systems.
  • Calibration routines are automated via software (Qiskit, Cirq, custom lab frameworks).
  • Goal: keep process infidelity \(\epsilon\) below fault-tolerance thresholds (\(\sim 10^{-3}\) today).
Key idea: Control theory meets experiment — model the qubit, measure errors, optimise pulses, verify with benchmarking, repeat.

Prerequisites: Qubits and the Bloch Sphere, Quantum Gates.

2) Qubit control Hamiltonian

A driven qubit in the rotating frame is described by:

\(H(t) = \frac{\hbar}{2}\bigl[\Delta(t)\,\sigma_z + \Omega(t)\bigl(\cos\phi(t)\,\sigma_x + \sin\phi(t)\,\sigma_y\bigr)\bigr]\)

where \(\Delta\) is the drive detuning, \(\Omega(t)\) the Rabi frequency envelope, and \(\phi(t)\) the drive phase. Choosing \(\Omega(t)\) and \(\phi(t)\) implements desired unitaries.

Rotating wave approximation: Near resonance (\(\Delta \ll \omega_d\)), fast oscillating terms average to zero, leaving the effective Hamiltonian above.

Open-system dynamics include Lindblad dissipators for relaxation and dephasing:

\(\dot{\rho} = -i[H,\rho] + \sum_k \mathcal{D}[L_k]\rho\)

4) Pulse shaping and DRAG

Square pulses cause spectral leakage to non-computational states. Derivative Removal by Adiabatic Gate (DRAG) adds an orthogonal quadrature proportional to the time derivative of the in-phase envelope:

\(\Omega_I(t) = A(t), \qquad \Omega_Q(t) = \alpha\,\frac{\dot{A}(t)}{\Delta}\)

where \(\alpha\) is a correction scale and \(\Delta\) the anharmonicity. DRAG suppresses \(|1\rangle \to |2\rangle\) leakage on transmon qubits.

Common envelopes

  • Gaussian: smooth, band-limited
  • Cosine: \(\cos^2\) rise/fall
  • Slepian / flat-top: sharp but controlled sidelobes

Advanced methods

  • GRAPE: gradient ascent pulse engineering
  • CRAB: chopped random basis optimisation
  • Model predictive control: feedback during gate

5) Randomized benchmarking (RB)

Randomized benchmarking estimates average gate fidelity over a Clifford group. Random sequences of \(m\) Clifford gates are followed by an inverting gate; survival probability decays as:

\(F(m) = A p^m + B\)

where \(p\) is the average fidelity per Clifford and \(A, B\) absorb state preparation and measurement (SPAM) errors. The average Clifford infidelity is \(\epsilon_C = 1 - p\).

Interleaved RB: Insert a target gate \(G\) between random Cliffords to estimate \(\epsilon_G\) specifically: compare decay rates with and without interleaving.

Extensions: simultaneous RB (crosstalk), cross-entropy benchmarking (XEB) for non-Clifford gates.

6) Two-qubit gate calibration

Calibrating entangling gates requires tuning coupling strength, interaction duration, and cancellation of parasitic single-qubit rotations. Typical workflow:

  1. Characterise coupling \(g\) or equivalent interaction rate.
  2. Measure conditional phase (e.g. CZ angle) vs pulse parameters.
  3. Optimise pulse shape to minimise leakage to non-computational states.
  4. Verify with quantum process tomography (QPT) or gate set tomography (GST).
  5. Monitor crosstalk on spectator qubits.
Common mistake: Calibrating single-qubit gates in isolation ignores simultaneous-drive crosstalk. Parallel RB reveals errors invisible in serial calibration.

Related: Multi-Qubit Systems, Superconducting Qubits, Trapped-Ion Quantum Systems.

7) Readout calibration

Dispersive readout assigns each measurement outcome a label (0 or 1) based on integrated IQ signal. Calibration steps:

  • Prepare \(|0\rangle\) and \(|1\rangle\) separately; record IQ clouds.
  • Fit Gaussian mixture models; set decision boundary (threshold line in IQ plane).
  • Measure assignment fidelity and readout-induced state collapse errors.
  • Apply measurement correction matrix to mitigate readout errors in post-processing.

Readout fidelity \(F_R\) and measurement operator \(M\) enter error-mitigation protocols. For \(N\) qubits, multi-qubit readout requires independent calibration per qubit plus crosstalk correction.

8) Automation and software stack

Modern labs run calibration DAGs (directed acyclic graphs) that execute spectroscopy → pulse update → RB verification in sequence, triggered on drift detection.

Tools

  • Qiskit Experiments / Ignis
  • pyGSTi (gate set tomography)
  • LabOne Q (Zurich Instruments)
  • Custom Python + AWG control

Drift triggers

  • Frequency shift > 1 MHz
  • RB fidelity drop > 0.1%
  • Temperature excursion
  • Scheduled time interval
Machine learning: Neural networks and reinforcement learning increasingly tune pulse parameters in real time, adapting to non-stationary noise.
