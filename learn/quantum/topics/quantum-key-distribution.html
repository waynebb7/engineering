1) What QKD provides

Quantum key distribution allows two parties (Alice and Bob) to establish a shared secret key with security guaranteed by the laws of quantum mechanics, not computational hardness. Any eavesdropper (Eve) attempting to learn the key inevitably introduces detectable disturbances.

  • Security is information-theoretic — unconditional against unlimited computing power.
  • QKD distributes keys; encryption uses standard symmetric ciphers (AES) with the shared key.
  • Does not by itself provide authentication — classical authentication is needed to prevent man-in-the-middle attacks.
  • Commercial systems: ID Quantique, Toshiba, QuantumCTek; standards: ETSI, ITU-T.
Key idea: Quantum states cannot be copied (No-Cloning Theorem) and measurement disturbs the state — Eve's presence is revealed through increased error rates.

Prerequisites: No-Cloning Theorem, Entanglement and Bell Inequalities.

2) BB84 protocol

Bennett and Brassard (1984) encode bits in photon polarisations across two conjugate bases:

BasisBit 0Bit 1
Rectilinear (+)\(|0\rangle \equiv |H\rangle\)\(|1\rangle \equiv |V\rangle\)
Diagonal (×)\(|0\rangle \equiv |D\rangle = \tfrac{1}{\sqrt{2}}(|H\rangle+|V\rangle)\)\(|1\rangle \equiv |A\rangle = \tfrac{1}{\sqrt{2}}(|H\rangle-|V\rangle)\)
  1. Alice randomly chooses basis and bit; sends encoded photons.
  2. Bob randomly chooses measurement basis and records outcomes.
  3. Public discussion: keep bits where bases matched; discard the rest.
  4. Estimate error rate \(Q\) on a sacrificed subset; if \(Q\) exceeds threshold, abort.
  5. Apply privacy amplification to distil a shorter, unconditionally secure key.

3) Security and the error threshold

For BB84 with single-photon sources, the secure key rate \(R\) (bits per pulse) depends on the quantum bit error rate (QBER) \(Q\). In the asymptotic limit with ideal single-photon sources:

\(R = 1 - h(Q) - Q\,h\!\left(\frac{f}{Q}\right)\)

where \(h(x) = -x\log_2 x - (1-x)\log_2(1-x)\) is the binary entropy and \(f\) is the error-correction efficiency. Security holds for \(Q \lesssim 11\%\) against individual attacks; improved proofs extend this.

Eavesdropping bound: In intercept-resend attacks, Eve's wrong-basis measurements cause \(\sim 25\%\) QBER — easily detected. Coherent attacks are handled by complete security proofs (Renner, Shor–Preskill).

4) E91 entanglement-based QKD

Ekert (1991) uses entangled photon pairs (e.g. Bell state \(|\Phi^+\rangle = \tfrac{1}{\sqrt{2}}(|HH\rangle + |VV\rangle)\)). Alice and Bob measure in random bases; correlations violate Bell inequalities when no eavesdropper is present.

The CHSH Bell parameter \(S\) satisfies \(|S| \leq 2\) classically but \(|S| \leq 2\sqrt{2}\) quantumly. Observing \(S > 2\) certifies entanglement and bounds Eve's information.

Device-independent QKD: If Bell violation is certified, security does not depend on device models — the strongest form of QKD security, demonstrated in laboratory settings.

Related: Entanglement and Bell Inequalities, Photonic Quantum Computing.

5) Practical challenges: losses and decoy states

Real fibres attenuate photons exponentially: transmission \(\eta = 10^{-\alpha L/10}\) for length \(L\) and loss \(\alpha\) (dB/km). At 100 km, \(\eta \sim 10^{-2}\) — most pulses are lost.

Multi-photon problem: Weak coherent pulses (WCP) sources occasionally emit two or more photons; Eve can perform photon-number splitting (PNS) attacks. The decoy-state method (Lo, Ma, Chen, 2005) inserts pulses of varying intensity so that multi-photon contributions can be bounded:

  • Signal states: mean photon number \(\mu \sim 0.5\).
  • Decoy states: weaker intensity \(\nu \sim 0.1\).
  • Vacuum states: estimate background counts.
Rate-distance trade-off: Secure key rate decreases exponentially with distance. Quantum repeaters (entanglement swapping) are needed for intercontinental QKD without trusted nodes.

6) QKD network architectures

Point-to-point

  • Direct fibre link between two parties
  • Range: ~100–150 km (without repeater)
  • Simplest deployment model

Trusted relay

  • Intermediate nodes regenerate keys
  • Extends range but trusts relay operators
  • Used in Chinese quantum backbone network

Satellite QKD: Micius satellite demonstrated intercontinental key distribution by reducing atmospheric path length. Free-space links connect ground stations via low-Earth-orbit relays.

MDI-QKD (measurement-device-independent): Alice and Bob send photons to a central untrusted measurement station; security does not depend on detector side-channel freeness.

7) Post-processing pipeline

  1. Sifting: retain matching-basis outcomes.
  2. Error estimation: compute QBER from test bits.
  3. Error correction: reconcile keys using Cascade, LDPC, or polar codes; leak \(\ell_{\text{EC}}\) bits classically.
  4. Privacy amplification: apply universal hash function to shorten key by \(\ell_{\text{PA}}\) bits, removing Eve's information.

Final secure key length (asymptotic, single-photon):

\(\ell_{\text{key}} = n\bigl[1 - h(Q) - f\,h(Q)\bigr] - \ell_{\text{EC}} - \ell_{\text{PA}}\)

Finite-size effects reduce key rates for short blocks; modern security proofs account for statistical fluctuations.

8) QKD vs post-quantum cryptography

PropertyQKDPost-quantum crypto (PQC)
Security basis Quantum physics (information-theoretic) Computational hardness (lattice, hash, etc.)
Infrastructure Dedicated optical links, specialised hardware Standard internet, software upgrade
Range Limited without repeaters Global (internet-scale)
Authentication Requires pre-shared classical key Built into protocol (signatures)

QKD and PQC are complementary: QKD protects key exchange on dedicated links; PQC protects general internet traffic against future quantum computers.
