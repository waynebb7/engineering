Grover's Search Algorithm

Quantum guide. Quadratic speedup for searching an unstructured database — amplitude amplification, the Grover iterate, and optimal query complexity.

← Quantum Subjects Catalog

1) The search problem

Search an unstructured database of \(N = 2^n\) items for one marked entry \(x^*\) where \(f(x^*) = 1\) and \(f(x) = 0\) elsewhere.

  • Classical: \(O(N)\) oracle queries in expectation (\(O(N)\) worst case)
  • Grover (1996): \(O(\sqrt{N})\) queries — quadratic speedup
Key idea: Grover is optimal for unstructured search — no quantum algorithm can do better than \(\Theta(\sqrt{N})\) queries.

Prerequisite: Quantum Circuits, Deutsch–Jozsa (oracle familiarity).

2) Initialisation and uniform superposition

Prepare \(n\) qubits in uniform superposition: \[ |s\rangle = H^{\otimes n}|0\rangle^{\otimes n} = \frac{1}{\sqrt{N}}\sum_{x=0}^{N-1}|x\rangle \]

The marked state \(|x^*\rangle\) has amplitude \(\frac{1}{\sqrt{N}}\) — finding it by measurement succeeds with probability \(\frac{1}{N}\), requiring \(O(N)\) repetitions.

Goal: Amplify the amplitude of \(|x^*\rangle\) to near 1 before measuring.

3) The oracle

The Grover oracle \(O_f\) flips the sign of the marked state: \[ O_f|x\rangle = \begin{cases}-|x\rangle & f(x) = 1\\ |x\rangle & f(x) = 0\end{cases} = \bigl(I - 2|x^*\rangle\langle x^*|\bigr) \]

Equivalently: \(O_f = I - 2\sum_{x: f(x)=1}|x\rangle\langle x|\). For one marked item, this is reflection about \(|x^*\rangle\).

Implementation: Phase oracle from Deutsch–Jozsa — same phase kickback trick with ancilla in \(|-\rangle\).

4) The diffusion operator

The diffusion operator (about the mean) reflects amplitudes about their average: \[ D = 2|s\rangle\langle s| - I \]

Constructed from \(H^{\otimes n}\), \(X^{\otimes n}\), and a multi-controlled \(Z\) (or \(T^{\otimes n}\) phase gate on \(|0\rangle^{\otimes n}\)): \[ D = H^{\otimes n}\bigl(2|0\rangle^{\otimes n}\langle 0|^{\otimes n} - I\bigr)H^{\otimes n} \]

Effect: Amplitudes above average get larger; below average get smaller — amplitude amplification.

5) The Grover iterate

One Grover iteration: \(G = D \cdot O_f\) (apply oracle, then diffusion).

In the 2D subspace spanned by \(|s\rangle\) and \(|x^*\rangle\), each iteration rotates the state by angle \(\theta \approx \frac{2}{\sqrt{N}}\) toward \(|x^*\rangle\).

Optimal number of iterations: \[ k \approx \frac{\pi}{4}\sqrt{N} \] Then measure — success probability \(\geq 1 - O(1/N)\).

6) Multiple marked items

If \(M\) items are marked (\(1 \leq M \leq N\)), the optimal iteration count is: \[ k \approx \frac{\pi}{4}\sqrt{\frac{N}{M}} \]

If \(M\) is unknown, quantum counting (using phase estimation on Grover) estimates \(M\) first.

Too many iterations: Over-rotating past \(|x^*\rangle\) reduces success probability — must tune \(k\).

7) Applications and limitations

  • NP search: Speeds up brute-force SAT, graph problems quadratically — not exponential.
  • Cryptography: Reduces effective key search space (symmetric keys need doubling in length).
  • Optimisation: Amplitude amplification generalises to quantum walk frameworks.
  • Limitation: Quadratic, not exponential — unstructured search has fundamental \(\Omega(\sqrt{N})\) quantum lower bound.

Compare: Shor's Algorithm achieves exponential speedup for factoring.
