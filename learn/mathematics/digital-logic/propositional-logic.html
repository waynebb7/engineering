1) Propositions and statements

A proposition (or statement) is a declarative sentence that is unambiguously true or false, but not both.

  • Proposition: "Paris is the capital of France." (true)
  • Proposition: 2 + 3 = 6 (false)
  • Not a proposition: "Close the door." (command)
  • Not a proposition: x + 1 = 5 (truth depends on x — this becomes a proposition only when x is fixed)

We use letters p, q, r for propositions. In digital circuits the same ideas appear as 0/1 signals — see Logic Gates.

Key idea: Propositional logic studies how truth values combine under connectives, not the internal meaning of atomic statements.

2) Basic connectives

Connectives build compound statements from simpler ones. The five standard connectives:

ConnectiveSymbolMeaningGate analogue
Negation¬pnot pNOT
Conjunctionp ∧ qp and qAND
Disjunctionp ∨ qp or q (inclusive)OR
Implicationp → qif p then q—
Biconditionalp ↔ qp if and only if qXNOR

Implication p → q is false only when p is true and q is false. When p is false, the implication is vacuously true regardless of q.

3) Truth tables for compound statements

A truth table lists every combination of input truth values and the resulting output. For two propositions there are 2² = 4 rows; for n variables, 2ⁿ rows.

4) Tautologies, contradictions, and contingencies

Classifications

  • Tautology: always true (e.g. p ∨ ¬p)
  • Contradiction: always false (e.g. p ∧ ¬p)
  • Contingency: true for some assignments, false for others

Important tautologies

  • Law of excluded middle: p ∨ ¬p
  • Law of non-contradiction: ¬(p ∧ ¬p)
  • Identity: p → p
Worked example: Show (p → q) ↔ (¬p ∨ q) is a tautology.
Build the truth table — the final column is all T regardless of p and q. This equivalence is central to rewriting implications in Boolean algebra.

5) Logical equivalence

Two formulas φ and ψ are logically equivalent (φ ≡ ψ) when they have identical truth tables. Common equivalences:

NameEquivalence
Double negation¬(¬p) ≡ p
De Morgan¬(p ∧ q) ≡ ¬p ∨ ¬q
De Morgan¬(p ∨ q) ≡ ¬p ∧ ¬q
Material implicationp → q ≡ ¬p ∨ q
Contrapositivep → q ≡ ¬q → ¬p
Distributivep ∧ (q ∨ r) ≡ (p ∧ q) ∨ (p ∧ r)

Set-theoretic form of De Morgan: Set Theory Basics. Gate-level form: DeMorgan's Theorems.

6) Natural deduction — introduction

Natural deduction is a proof system where you derive conclusions from premises using introduction and elimination rules for each connective. Proofs are trees (or sequences) of justified lines.

Core rules (summary)

  • ∧-intro: from p and q, infer p ∧ q
  • ∧-elim: from p ∧ q, infer p (or q)
  • ∨-intro: from p, infer p ∨ q
  • →-intro: assume p, derive q, discharge to get p → q
  • →-elim (modus ponens): from p and p → q, infer q
  • ¬-intro: assume p, derive ⊥, discharge to get ¬p

Example proof

Prove p → p:

  1. Assume p
  2. Reiterate p (from assumption)
  3. Discharge assumption → p → p by →-intro
Valid argument forms: Modus ponens (p, p→q ⊢ q) and modus tollens (¬q, p→q ⊢ ¬p) are sound. Affirming the consequent (q, p→q ⊢ p) is not valid.

7) Worked examples

Example A — simplify: ¬(p → q) ∨ q
p → q ≡ ¬p ∨ q, so ¬(¬p ∨ q) ∨ q ≡ (p ∧ ¬q) ∨ q ≡ p ∨ q by absorption.
Example B — prove tautology: ⊢ (p ∧ q) → p
Assume p ∧ q. By ∧-elim, obtain p. Discharge → (p ∧ q) → p.
Example C — translate: "If the alarm sounds, then the door is open or the sensor is faulty."
A → (D ∨ F) where A = alarm sounds, D = door open, F = sensor faulty.

8) Quick knowledge check

  1. Which connective matches the NAND gate output for p, q?
  2. Is (p → q) ∧ (q → p) equivalent to p ↔ q?
  3. Classify p ∧ ¬p: tautology, contradiction, or contingency?
  4. Write p → q using only ¬ and ∨.
  5. Translate: "It is not the case that both switches are on."
  6. Using modus ponens, what follows from premises p → q and p?
Show answers
  1. (1) ¬(p ∧ q)
  2. (2) yes
  3. (3) contradiction
  4. (4) ¬p ∨ q
  5. (5) ¬(S₁ ∧ S₂) or ¬S₁ ∨ ¬S₂
  6. (6) q.

Answers: (1) ¬(p ∧ q); (2) yes; (3) contradiction; (4) ¬p ∨ q; (5) ¬(S₁ ∧ S₂) or ¬S₁ ∨ ¬S₂; (6) q.
