1) Positional notation

A base (or radix) tells you how many distinct digits are used before place value increases. In base b, each position is worth b times the position to its right.

  • Decimal (base 10): digits 0–9 — everyday counting
  • Binary (base 2): digits 0, 1 — the language of digital circuits
  • Octal (base 8): digits 0–7 — compact grouping of binary bits
  • Hexadecimal (base 16): digits 0–9 and A–F — common in programming and debuggers
Key idea: The value of a number does not change when you rewrite it in another base — only the symbols change. 1510 = 11112 = 178 = F16.

See the topic overview at Digital Math Classification. Conversions feed directly into Binary Arithmetic.

2) Decimal to binary

To convert a whole decimal number to binary, repeatedly divide by 2 and record remainders. Read remainders from bottom to top.

Worked example: Convert 45 to binary.
45 ÷ 2 = 22 r 1 · 22 ÷ 2 = 11 r 0 · 11 ÷ 2 = 5 r 1 · 5 ÷ 2 = 2 r 1 · 2 ÷ 2 = 1 r 0 · 1 ÷ 2 = 0 r 1
Remainders (bottom up): 1011012

Alternative — subtract powers of 2

Largest power ≤ 45 is 32. 45 − 32 = 13.

Next: 8 → 13 − 8 = 5. Then 4 → 5 − 4 = 1. Then 1.

Powers used: 32, 8, 4, 1 → bits at positions 5,3,2,0 → 101101

Check

32 + 8 + 4 + 1 = 45 ✓

Always verify by converting back to decimal.

3) Binary to decimal

Multiply each bit by its place value (powers of 2 from right to left, starting at 2⁰ = 1) and add.

Worked example: 1101012
= 1×32 + 1×16 + 0×8 + 1×4 + 0×2 + 1×1 = 32 + 16 + 4 + 1 = 5310
Bit position (power) 5 (32) 4 (16) 3 (8) 2 (4) 1 (2) 0 (1)
Bit value 1 1 0 1 0 1
Contribution 32 16 0 4 0 1

4) Binary ↔ octal and hex

Because 8 = 2³ and 16 = 2⁴, binary converts to octal in groups of 3 bits and to hex in groups of 4 bits, starting from the right. Pad with leading zeros if needed.

Binary to hex

1011 11002 → 1011=B, 1100=C → BC16

0010 11012 → 2D16

Binary to octal

10 111 1002 → 2, 7, 4 → 2748

Common mistake: Grouping from the left instead of the right. Always split from the least significant bit upward, then pad zeros on the left of the highest group only.

5) Decimal ↔ hex and octal

For bases that are not powers of 2, convert through binary or use repeated division by the target base.

Decimal to hex: Divide by 16 repeatedly; remainders 10–15 become A–F.
Example: 255 ÷ 16 = 15 r 15 → FF16
Hex to decimal: Expand place values as powers of 16.
Example: 2A16 = 2×16 + 10 = 4210

Via binary (reliable)

  1. Decimal → binary (divide by 2)
  2. Binary → hex/octal (group bits)

Direct (faster when confident)

  1. Decimal ↔ hex: divide/multiply by 16
  2. Decimal ↔ octal: divide/multiply by 8

6) Fractional values (brief)

For fractional parts, use negative powers of the base. Binary fractions use halves, quarters, eighths, and so on.

Example: 0.1012 = ½ + 0 + ¼ = 0.62510

Some decimal fractions have no exact finite binary representation (e.g. 0.110). This matters in floating-point hardware — see Binary Arithmetic for integer work first.

7) Why hex and octal matter in practice

  • Memory addresses — shown as hex in debuggers (0x00401000)
  • Colour codes — RGB as 24-bit hex (#FF5733)
  • Byte-oriented data — two hex digits = one byte (8 bits)
  • Unix permissions — octal (chmod 755)
Engineering habit: Learn the one-byte hex table (00–FF) and the 4-bit binary ↔ hex nibble map — it saves time when reading register dumps alongside Truth Tables and Boolean Algebra bit masks.
