1) Binary addition rules

Binary addition uses the same column method as decimal, but each column sums only 0 and 1. When the sum is 2 or more, carry 1 to the next column.

A B Sum Carry out
0000
0110
1010
1101
Key idea: 1 + 1 = 102 — write 0 in the column, carry 1 left. Full adder circuits in Combinational Logic implement this bit-by-bit.

Assumes you can convert bases — see Number Systems.

2) Adding multi-bit binary numbers

Align bits by place value, add from right to left, propagating carries.

Worked example: 10112 + 01102
Column 0: 1+0=1 · Column 1: 1+1=0 carry 1 · Column 2: 0+1+1=0 carry 1 · Column 3: 1+0+1=0 carry 1
Result: 100012 (17 in decimal: 11 + 6)

Column layout

+ 0110
------
 10001

Check in decimal

11 + 6 = 17 ✓

Convert back if unsure: 100012 = 17

3) Binary subtraction

Subtract column by column. If the top bit is smaller, borrow 2 from the next column (because each place is worth 2, not 10).

Worked example: 11012 − 01012
= 10002 (13 − 5 = 8)
Rule Result
0 − 00
1 − 01
1 − 10
0 − 11 with borrow from next column
Common mistake: Borrowing 10 instead of 2. In binary, a borrow adds 2 to the current column, not 10.

4) Unsigned vs signed numbers

A fixed bit width (e.g. 8 bits) can represent either:

  • Unsigned — all bits contribute to magnitude (0 to 255 for 8 bits)
  • Signed (two's complement) — the leftmost bit indicates sign; negative values use a complement scheme

8-bit unsigned range

00000000 = 0

11111111 = 255

8-bit signed range

10000000 = −128

01111111 = +127

Why it matters: CPUs use two's complement for integer ALU operations. The same bit pattern means different values depending on whether the operation is signed or unsigned.

5) Two's complement representation

To write a negative number in n bits:

  1. Write the positive number in binary (n bits)
  2. Flip every bit (one's complement)
  3. Add 1
Worked example: −5 in 8 bits
+5 = 00000101 → flip → 11111010 → add 1 → 11111011
Quick test: The leftmost bit is 1 for negative numbers (for non-zero values in normal range). 11111011 as signed 8-bit = −5.

6) Subtraction via two's complement

To compute A − B, add A to the two's complement of B. Discard any carry beyond the bit width — the remaining bits are the result.

Worked example (4-bit): 0101 − 0011 (5 − 3)
Two's complement of 0011: flip → 1100, +1 → 1101
0101 + 1101 = 10010 → drop carry → 0010 = 2 ✓
Overflow: If adding two positive numbers gives a negative result (or vice versa), the true result does not fit in n bits — an overflow flag is set in hardware.

This is the same bit-level logic used when simplifying expressions in Boolean Algebra and verifying with Truth Tables.

7) Multiplication and shift (overview)

Multiplying by 2k in binary is a left shift by k places; dividing by 2k is a right shift. Full multiplication is repeated add-and-shift — implemented in ALUs and studied further in Combinational Logic.

Left shift

00000101 (5) → << 1 → 00001010 (10)

Right shift

00001010 (10) → >> 1 → 00000101 (5)
