Truth Functional Completeness

Digital logic guide. Functional completeness, building any gate from NAND alone, and a preview of disjunctive and conjunctive normal forms.

← Pure Math Subjects Catalog

1) Truth functions and connectives

A truth function maps finitely many Boolean inputs to a single Boolean output. With n variables there are 2^(2ⁿ) distinct truth functions. For n = 2 inputs, that is 16 functions — including AND, OR, XOR, and NAND.

A connective (or gate) implements a truth function. Truth tables completely specify any truth function.

Key idea: "Truth-functional" means the output depends only on the current input values, not on history or timing — the model of combinational logic.

Background: Propositional Logic and Logic Gates.

2) Functional completeness defined

A set of connectives S is functionally complete (adequate, universal) if every truth function can be expressed using only connectives from S (with repetition allowed).

  • {¬, ∧, ∨} — the classical trio — is functionally complete
  • {¬, ∧} is complete (can define OR via De Morgan)
  • {¬, ∨} is complete
  • {NAND} alone is complete
  • {NOR} alone is complete
Not complete alone: {AND, OR} without NOT cannot express negation — any formula built from them is monotone (output flips 0→1 only when inputs do), so XOR is unreachable.

3) NAND is functionally complete

The NAND gate (↑, p | q, or ¬(p ∧ q)) is the smallest universal building block in digital electronics. Three identities prove adequacy:

Fundamental NAND constructions:
NOT p ≡ p NAND p
p AND q ≡ (p NAND q) NAND (p NAND q)
p OR q ≡ (p NAND p) NAND (q NAND q)

Once NOT, AND, and OR are available, any truth table column can be built — hence NAND alone suffices. The same argument works for NOR with dual constructions.

Worked example: Build XOR from NAND only:
p ⊕ q = (p NAND (p NAND q)) NAND (q NAND (p NAND q))

4) NOR is functionally complete

NOR (↓, ¬(p ∨ q)) is the dual universal gate:

NOT p ≡ p NOR p
p OR q ≡ (p NOR p) NOR (q NOR q)
p AND q ≡ (p NOR p) NOR (q NOR q) then NOR'd — equivalently (p NOR q) NOR (p NOR q) after building NOT

Practical note: CMOS NAND and NOR layouts are both efficient; chip families (74xx series) provide all primitives, but synthesis tools often map designs to NAND/NOR networks for area optimisation.

De Morgan bridge: DeMorgan's Theorems.

5) Building NOT, AND, OR from NAND — circuit view

NOT from NAND

Tie both inputs together: one NAND with A on both pins gives A'.

AND from NAND

NAND the inputs, then NAND the result with itself (invert the NAND output).

OR from NAND:
Step 1: Invert each input: A' = A NAND A, B' = B NAND B.
Step 2: A OR B = A' NAND B' (by De Morgan: (A' NAND B') = (A' ∧ B')' = A ∨ B).

This is why a chip containing only NAND gates can implement any combinational circuit.

6) DNF preview — disjunctive normal form

Disjunctive Normal Form (DNF) — also called sum of products (SOP) — is an OR of AND terms (minterms). Every truth function has a DNF representation.

Method: List rows of the truth table where output = 1. For each row, AND the literals (variable or its negation) that match that row. OR all such product terms together.
Worked example: F(A,B) = A ⊕ B
Minterms where F=1: row 01 → A'·B; row 10 → A·B'.
DNF: F = A'·B + A·B'

DNF is canonical but often redundant — simplify with Boolean Algebra, Karnaugh Maps, or Quine–McCluskey.

7) CNF preview — conjunctive normal form

Conjunctive Normal Form (CNF) — product of sums (POS) — is an AND of OR terms (maxterms). Dual to DNF: use rows where output = 0.

Method: For each row where F = 0, form a sum term that is 0 on that row (use the complementary literals). AND all maxterms together.
Worked example: F(A,B) = A ⊕ B
Rows where F=0: 00 → maxterm (A + B); 11 → (A' + B').
CNF: F = (A + B) · (A' + B')

CNF is central in automated reasoning (SAT solvers) and in deriving NAND-only networks via De Morgan.

8) Engineering implications

  • Standard-cell libraries can be built around one primitive (NAND) — simplifies manufacturing and layout rules.
  • Logic synthesis converts HDL descriptions to NAND/NOR netlists automatically.
  • DNF → AND-OR circuits; CNF → OR-AND circuits — direct mapping to two-level gate networks.
  • Functional completeness guarantees that no "missing gate" blocks a design — any Boolean spec is realisable.
Quick knowledge check:
  1. How many distinct 2-input truth functions exist?
  2. Express NOT A using only NAND.
  3. Write DNF for F = A · B (2 variables).
  4. Why is {AND, OR} alone not functionally complete?
Answers: (1) 16; (2) A NAND A; (3) A · B (single minterm); (4) cannot express negation/inversion.
