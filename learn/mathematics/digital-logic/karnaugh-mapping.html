1) What is a Karnaugh map?

A Karnaugh map (K-map) is a grid layout of a truth table arranged so that adjacent cells differ in exactly one input variable (Gray-code order). That adjacency makes redundant literals visible as rectangular groups.

  • Purpose: simplify sum-of-products or product-of-sums expressions
  • Alternative to long algebraic manipulation with Boolean algebra
  • Best suited to hand minimisation of functions with up to about six variables
Key idea: Grouping adjacent 1s (or 0s) merges minterms and eliminates variables that change within the group.

2) Two-variable K-map

For F(A, B), the map has four cells. Row/column labels use Gray code so neighbours differ by one bit.

B = 0B = 1
A = 0m₀ (A'B')m₁ (A'B)
A = 1m₂ (AB')m₃ (AB)

Example: F = Σ(1, 3) — 1s in the entire right column.

Group the two 1s → B changes within the group, A is redundant → F = B.

Edges wrap: in a 2×2 map, all four corners are mutually adjacent (one group of four gives F = 1).

3) Three-variable K-map

A 3-variable map is 2 rows × 4 columns (8 cells). One variable (often A) labels the rows; BC labels columns in Gray order: 00, 01, 11, 10.

A \ BC00011110
0m₀m₁m₃m₂
1m₄m₅m₇m₆

Example: F = Σ(1, 3, 5, 7) — all 1s in columns where C = 1.

One group of four 1s spans both rows → F = C.

Left and right column edges are adjacent (wrap-around), like a cylinder.

4) Four-variable K-map

A 4-variable map is 4 × 4 (16 cells). Rows use Gray-ordered AB; columns use Gray-ordered CD.

AB \ CD00011110
00m₀m₁m₃m₂
01m₄m₅m₇m₆
11m₁₂m₁₃m₁₅m₁₄
10m₈m₉m₁₁m₁₀

Adjacency wraps on both horizontal and vertical edges, and at corners — the map behaves like a torus. Groups can be 1, 2, 4, 8, or 16 cells.

Example: F = Σ(0, 2, 8, 10) — four corners are all 1 → single group → F = B'D'.

5) Grouping rules

Follow these rules when circling 1s for SOP minimisation (use 0s instead for POS):

  1. Groups must contain 1, 2, 4, 8, … cells (powers of two only)
  2. Groups must be rectangular (width and height are powers of two)
  3. Make each group as large as possible
  4. Cover every 1 at least once; use the fewest groups needed
  5. A group of 2k cells eliminates k variables from the product term
  6. Edge and corner wrap-around counts as adjacency
Common mistake: Making groups of three or six cells. Invalid groups do not correspond to legal product terms.

Each group yields one implicant (product term). The OR of all group terms is the minimised SOP.

6) Don't cares (X)

A don't care condition marks input combinations that never occur or whose output is irrelevant. On a K-map, don't cares are written as X and may be treated as 0 or 1 — whichever yields larger groups.

Example: BCD-to-7-segment displays: invalid codes 10–15 are don't cares.

  • Include X in a group only when it helps enlarge a valid group of 1s
  • Never create a group consisting only of Xs with no required 1
  • Don't cares can reduce term count but the simplified function is valid only for specified care conditions
Notation: F = Σ(1, 3) + d(2, 5) means minterms 1 and 3 are required 1s; 2 and 5 are don't cares.

7) Worked example (4 variables)

Simplify F(A, B, C, D) = Σ(1, 3, 7, 11, 15) + d(0, 2, 5).

Place 1s at the listed minterms; mark X at 0, 2, 5. One possible grouping:

  • Group of 4 covering m₀, m₁, m₃, m₂ (use X at m₀, m₂) → term B'D
  • Group of 4 covering m₇, m₅, m₁₅, m₁₃ (use X at m₅) → term BD
  • Group of 2 covering m₁₁, m₁₅ → term ACD

Minimised result: F = B'D + BD + ACD (other minimal covers may exist — compare group choices).

For systematic multi-output or large-variable minimisation, use the Quine–McCluskey method.

8) SOP vs POS on K-maps

SOP: group 1s, write product terms for each group, OR them together.

POS: group 0s instead, write sum terms for each group, AND them together. Complementing the map of 0s gives the dual form.

Choose whichever form needs fewer literals for your gate library. NAND-only designs often start from SOP then apply De Morgan's theorems.
