1) What is combinational logic?

A combinational circuit has outputs that depend only on the present input values. There is no memory: no flip-flops, no feedback that stores state across clock cycles.

  • Examples: adders, comparators, decoders, multiplexers, arbitrary Boolean functions
  • Contrast with sequential logic, where outputs also depend on stored state
  • Built from logic gates wired according to a Boolean expression or truth table
Key idea: Given the same inputs, a combinational circuit always produces the same outputs — after propagation delay settles.

2) Minterms and maxterms

For n input variables there are 2ⁿ input combinations. Each combination maps to a unique minterm (product of literals) or maxterm (sum of literals).

A function is often written in canonical form: F = Σ m(1, 3) (sum of minterms) or F = Π M(0, 2) (product of maxterms).

3) Sum of Products (SOP)

Sum of Products expresses a function as an OR of AND terms. Each AND term is a product term covering one or more minterms.

Example: F(A, B, C) = Σ(1, 3, 7)

  • m₁ = A'B'C
  • m₃ = A'BC
  • m₇ = ABC

Canonical SOP: F = A'B'C + A'BC + ABC

After simplification (using Boolean algebra or a K-map): F = A'C + AB — fewer gates, same function.

Circuit mapping: Each product term → AND gate; the OR of terms → OR gate. Inverted inputs supply literals.

4) Product of Sums (POS)

Product of Sums is the dual form: an AND of OR terms (sum terms), often derived from maxterms where F = 0.

Example: for F = Σ(1, 3, 7), the zeros are at m₀, m₂, m₄, m₅, m₆:

F = (A + B + C)(A + B' + C)(A' + B + C')(A' + B' + C)(A' + B' + C')

SOP is usually preferred for two-level AND-OR implementations. POS suits NAND-NAND or NOR-NOR structures and complements derived from De Morgan's theorems.

5) Combinational design flow

Standard procedure from specification to gate-level circuit:

  1. Specification — define inputs, outputs, and behaviour (word problem, timing diagram, or interface)
  2. Truth table — list all input combinations and required outputs; see Truth Tables
  3. Boolean expression — write SOP or POS from the truth table
  4. Simplification — reduce terms using algebra, K-maps, or Quine–McCluskey
  5. Implementation — draw the logic diagram with chosen gate types (AND/OR, NAND-only, etc.)
  6. Verification — simulate or build; check all rows of the truth table and timing behaviour
Design tip: Simplify before drawing the circuit. A minimal SOP reduces chip count, power, and propagation delay.

7) Propagation delay

Real gates are not instantaneous. Each gate adds a propagation delay tpd; the longest path from input change to stable output is the critical path.

  • More logic levels → longer worst-case delay
  • Simpler expressions often mean faster circuits
  • Delay matters in high-speed arithmetic, bus decoding, and when feeding sequential elements

Combinational delay is one reason simplified SOP/POS forms matter in practice — not just gate count.

8) Introduction to hazards

A hazard is an unwanted brief glitch on an output when inputs change, even though the steady-state truth table is correct. Hazards arise from unequal propagation paths through the logic.

Type Description
Static-1 hazard Output should stay 1 but momentarily dips to 0
Static-0 hazard Output should stay 0 but momentarily spikes to 1
Dynamic hazard Output should make a single transition but oscillates multiple times

Example: In F = AB + A'C, when B = C = 1 and A changes, both product terms can momentarily be 0 before the new value propagates — a static-1 hazard.

Mitigation: add redundant product terms (consensus terms), use hazard-free two-level covers, or register outputs with clocked flip-flops.
