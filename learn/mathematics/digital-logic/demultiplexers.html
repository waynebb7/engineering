Demultiplexers

Digital logic guide. Demultiplexer function, signal routing, and address-decoding applications.

← Pure Math Subjects Catalog

1) What is a demultiplexer?

A demultiplexer (DEMUX) takes one data input and routes it to exactly one of several outputs, as determined by select (address) lines. It is the inverse of a multiplexer.

  • Data input: D — single source signal
  • Select lines: S₀, S₁, … — binary address of the active output
  • Outputs: Y₀, Y₁, …, Yn−1 — exactly one carries D; others are 0 (or high-Z with tri-state)
Key idea: DEMUX distributes one signal to one of many destinations — "demux" = one-to-many routing.

2) DEMUX function

For a 1-to-2m DEMUX, output Yi is active when the select value equals i:

Yi = D · mi(S)

Each output is the data input ANDed with the minterm of the select variables for that output index. All other outputs are 0.

S₁S₀Y₀Y₁Y₂Y₃
00D000
010D00
1000D0
11000D

A DEMUX with an enable input implements Yi = EN · D · mi(S).

3) One-to-two and one-to-four DEMUX

1:2 DEMUX (one select line):

  • Y₀ = S'D
  • Y₁ = SD

1:4 DEMUX (two select lines):

  • Y₀ = S₁'S₀'D
  • Y₁ = S₁'S₀D
  • Y₂ = S₁S₀'D
  • Y₃ = S₁S₀D

Symbol labels: 1:2, 1:4, 1:8. A 1:8 DEMUX (e.g. 74138 family when used with data gating) requires three select lines.

4) Routing and signal distribution

Primary uses of DEMUX routing:

  • Serial-to-parallel: time-multiplex one wire into multiple parallel lines over successive cycles
  • Clock distribution: enable one branch of a clock tree (with care for skew)
  • Test access: route a test signal to one of several circuit nodes
  • LED/matrix drivers: steer current to one column or row at a time

Combined with a MUX, a DEMUX can form a bidirectional routing path: MUX concentrates many lines to one; DEMUX spreads one line back to many.

Contrast with decoder: A decoder activates one output line without carrying arbitrary data D — it generates address patterns. A DEMUX gates a data signal onto the selected output.

5) Address decoding

DEMUX structure is the core of address decoding in memory and I/O systems. A binary address on select lines activates one chip-select or memory location.

Example: 4-byte memory bank with 2-bit address:

  1. Address bus drives DEMUX select inputs
  2. Data input carries write data (or read enable)
  3. Each Yi connects to one register's clock or write-enable

Full memory decoders often use a decoder plus AND gates: CSi = EN · mi(Address). The 74138 3-to-8 decoder is effectively a DEMUX with inverted active-low outputs when G1, G2A, G2B enables are used.

Engineering note: Real systems add glue logic for address ranges, byte lanes, and bus contention — the DEMUX is the conceptual model.

6) Gate-level implementation

1:4 DEMUX from basic gates:

  1. Inverters generate S₁', S₀' if needed
  2. Four 3-input AND gates: each AND combines D with one select minterm
  3. Outputs drive destination lines directly

Equivalently: use a 2-to-4 decoder for select minterms, then AND each decoder output with D. This separates address decoding from data gating — a common PCB and FPGA pattern.

See Logic Gates and De Morgan's Theorems for NAND-only variants.

7) DEMUX as a decoder

If D = 1 (logic high), the DEMUX outputs are exactly the minterms of the select variables — it behaves as a binary decoder.

  • 1:4 DEMUX with D = 1 → 2-to-4 decoder outputs
  • With D = 0, all outputs are 0
  • With arbitrary D, the active output carries the data bit — conditional write/strobe

This duality explains why decoder IC datasheets describe "demultiplexing" and "decoding" interchangeably. Encoders perform the inverse information flow: many lines to few address bits.

8) Cascading and expansion

Build larger DEMUXes hierarchically:

  • Two 1:4 DEMUXes with a 1:2 first stage → 1:8 routing
  • Enable inputs disable entire subtrees — useful in memory banking
  • Tree depth adds propagation delay; balance select and data timing in synchronous designs
