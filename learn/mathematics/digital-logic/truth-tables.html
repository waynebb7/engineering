1) What is a truth table?

A truth table lists every possible combination of input values and shows the resulting output for a logic function. With n binary inputs there are 2n rows.

  • Inputs in columns (A, B, C, …)
  • Output column(s) for the function F
  • Rows cover 00…0 through 11…1 in binary counting order (usually)
Key idea: Truth tables are the ground truth for any gate or expression. If algebra and table disagree, the table wins — use it to verify simplifications in Boolean Algebra.

NAND / NOR

NAND = NOT(A AND B) — 0 only when both inputs are 1

NOR = NOT(A OR B) — 1 only when both inputs are 0

3) Building a table from scratch

For n inputs, follow this procedure:

  1. Write input columns; fill with binary count from 0 to 2n − 1
  2. Add intermediate columns for sub-expressions if helpful
  3. Compute the final output column one row at a time
Worked example: F = A AND (NOT B) — two inputs, four rows.
Row 0: A=0,B=0 → NOT B=1 → F=0 · Row 1: A=0,B=1 → F=0
Row 2: A=1,B=0 → F=1 · Row 3: A=1,B=1 → F=0
Output column: 0, 0, 1, 0

4) Truth table from a Boolean expression

Given F = AB' + A'B (XOR), build columns for AB', A'B, then OR them.

A B B' AB' A'B F
001000
010011
101101
110000
Order of operations: NOT first, then AND, then OR — same precedence as in Boolean Algebra. Use parentheses when in doubt.

5) From truth table to expression

To derive a sum-of-products expression, OR together a minterm for each row where F = 1.

Example: F = 1 on rows (0,1) and (1,0) for inputs A,B.
Row (0,1): A'B · Row (1,0): AB' · F = A'B + AB' (XOR)

Minterm notation

m0 = A'B', m1 = A'B, m2 = AB', m3 = AB

XOR = m1 + m2

Don't-care rows

X or − means output is unused — exploited in Karnaugh Mapping.

6) Three or more variables

Three inputs → 8 rows. Binary count order: 000, 001, 010, 011, 100, 101, 110, 111.

Tip: Gray-code column ordering appears in K-maps; standard truth tables use binary count. Gray Code differs by one bit per step — useful for encoders, not required here.

For half and full adders, truth tables define sum and carry outputs — connect to Binary Arithmetic.

7) Equivalence and verification

Two expressions are equivalent if their truth tables match in every row. Use this to prove laws such as De Morgan's — see De Morgan's Theorems.

Verify: (AB)' versus A' + B' — build both columns; they should be identical for all four rows.
Exam tip: When simplifying, always spot-check at least two rows (e.g. all-0 and all-1 inputs) against the original table.
