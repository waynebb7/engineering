1) Combinational vs sequential logic

Combinational circuits (see Combinational Logic) produce outputs that depend only on present inputs. Sequential circuits add memory: outputs depend on present inputs and past history.

  • State: the stored information (usually a register of flip-flops)
  • Next state: computed from current state and inputs
  • Clock: synchronises state updates in synchronous designs
Why memory matters: Counters, registers, state machines, and CPUs all require elements that hold values between clock cycles.

2) Latches vs flip-flops

Latch (level-sensitive)

Transparent while enable is active. Output tracks input during the enable window. Simple but timing-sensitive.

Flip-flop (edge-triggered)

Samples input only on a clock edge (rising or falling). Holds value between edges. Preferred in synchronous systems.

FeatureLatchFlip-flop
SensitivityLevel (enable high/low)Edge (clock transition)
TransparencyYes, when enabledNo — edge sample only
Typical useAsync interfaces, D-latch in some FFsRegisters, counters, FSMs
Timing analysisHarder (enable width matters)Cleaner (setup/hold at edge)
Common mistake: Using latches in a design intended to be fully synchronous — synthesis tools may infer unintended latches from incomplete if/else logic in HDL.

3) SR latch and flip-flop

The SR (Set-Reset) element has two inputs: Set forces output \(Q = 1\); Reset forces \(Q = 0\).

SRQ (after settle)Comment
00HoldNo change
010Reset
101Set
11InvalidForbidden for basic NOR/NAND SR latch

An SR flip-flop adds a clock: S and R are gated so they only affect \(Q\) on the active clock edge. The forbidden state is avoided by design in clocked versions.

4) D latch and D flip-flop

The D (Data) element eliminates the forbidden SR state by routing one input: \(D = 1\) sets \(Q = 1\); \(D = 0\) resets \(Q = 0\).

  • D latch: while enable (E) is high, \(Q\) follows \(D\); when E drops, \(Q\) holds the last value
  • D flip-flop: on the rising clock edge, \(Q \leftarrow D\); between edges, \(Q\) is stable
Most common element: D flip-flops form the basis of shift registers, pipeline stages, and the state registers in Finite State Machines.

Characteristic equation (edge-triggered D FF): \(Q_{next} = D\).

5) JK and T flip-flops

JK flip-flop

Generalises SR. J=1,K=0 sets; J=0,K=1 resets; J=K=0 holds; J=K=1 toggles \(Q\).

Equation: \(Q_{next} = J\overline{Q} + \overline{K}Q\)

T flip-flop

Toggle: if T=1, \(Q\) flips; if T=0, \(Q\) holds. Built from JK with J=K=T.

Equation: \(Q_{next} = T \oplus Q\)

T flip-flops simplify binary counters: connect T=1 on all stages and chain toggle outputs to higher bits.

6) Timing diagrams

A timing diagram plots signals against time. For sequential circuits, focus on the relationship between clock, data, and output.

  • Setup time (\(t_{su}\)): data must be stable before the clock edge
  • Hold time (\(t_h\)): data must remain stable after the clock edge
  • Clock-to-Q delay (\(t_{cq}\)): time from edge until output changes
  • Propagation delay: total delay through combinational logic between register stages
CLK  ‾|_|‾|_|‾|_|‾|_|‾
D    ‾‾‾\___/‾‾‾‾\___
Q         ‾‾‾\___/‾‾   (D FF: Q updates on rising edge, after t_cq)
Reading the diagram: Draw vertical lines at each rising clock edge. Check that D is stable in the setup/hold window around each edge. Q changes shortly after the edge, not instantaneously.
Metastability: If setup or hold is violated, the flip-flop may enter an indeterminate state for an unpredictable time. Always respect timing constraints in real designs.

7) Registers and shift registers

A register is a bank of flip-flops storing a multi-bit word, updated on each clock edge. A shift register connects flip-flops in series so bits move one position per clock:

  • SIPO: serial in, parallel out — useful for expanding one wire to many
  • PISO: parallel in, serial out — useful for serial transmission
  • Universal shift register: configurable direction and load mode

Shift registers underpin serial communication (SPI, UART) and arithmetic shift operations in processors.
