Multiplexers

Digital logic guide. Multiplexer function, gate-level implementations, and using a MUX as a Boolean function generator.

← Pure Math Subjects Catalog

1) What is a multiplexer?

A multiplexer (MUX) selects one of several data inputs and routes it to a single output. Think of it as a digitally controlled switch or "data selector."

  • Data inputs: I₀, I₁, …, In−1
  • Select lines: S₀, S₁, … — binary address of the chosen input
  • Output: Y — equals the selected data input
  • An n-input MUX needs ⌈log₂ n⌉ select lines
Key idea: A MUX implements the function Y = Ik where k is the binary value on the select bus.

2) MUX function

For a 2m-to-1 MUX with select Sm−1…S₀, the output is:

Y = Σ Ii · mi(S)

Each minterm mi of the select variables ANDs with data input Ii; the OR of all such terms forms the output.

S₁S₀Y (4:1 MUX)
00I₀
01I₁
10I₂
11I₃

Symbol blocks are labelled 2:1, 4:1, 8:1, etc. Enable inputs (when present) gate the output to high-impedance or force 0.

3) Two-to-one multiplexer

The simplest MUX has two data inputs and one select line.

Boolean form: Y = S'I₀ + SI₁

  • When S = 0: Y = I₀
  • When S = 1: Y = I₁

A 2:1 MUX is universal for combinational logic: with the right constants on I₀ and I₁, it can realise NOT, AND, and OR — see Truth Functional Completeness.

Building block: Larger MUXes are built by cascading 2:1 stages or by AND-OR structures with decoded selects.

4) Four-to-one and n-to-one MUX

4:1 MUX Boolean equation:

Y = S₁'S₀'I₀ + S₁'S₀I₁ + S₁S₀'I₂ + S₁S₀I₃

Structure:

  1. Decoder or AND gates generate minterms of S₁S₀
  2. Each minterm ANDs with the corresponding data input
  3. OR gate combines all product terms

8:1 MUX needs three select lines; 16:1 needs four. IC packages such as 74151 (8:1) and 74153 (dual 4:1) are common in lab work.

5) Gate-level implementation

Standard two-level realisation of a 4:1 MUX:

  • Inverters on select lines (if only true and complement are needed)
  • Four 3-input AND gates: S₁'S₀'I₀, S₁'S₀I₁, S₁S₀'I₂, S₁S₀I₃
  • One 4-input OR gate for Y

Alternatively, use a 2-to-4 decoder with active-high outputs feeding AND gates — the decoder supplies the select minterms. See Logic Gates for gate symbols.

Timing: Select changes can cause brief glitches on Y if data inputs differ — relevant when MUX feeds clocked logic.

6) MUX as a function generator

Any Boolean function of n variables can be implemented with a 2n-to-1 MUX by wiring constants and variables to the data inputs.

Procedure:

  1. Write the truth table for F(A, B, C, …)
  2. Use all but one variable as MUX select lines (usually the most significant variables)
  3. The remaining variable(s) appear on data inputs — each row of the truth table sets one Ii to 0, 1, or a literal

Example: F(A, B, C) = Σ(1, 2, 6, 7) using an 8:1 MUX with A, B as select:

ABIk connection
000
01C
10C'
111

A 4:1 MUX with two select lines can generate any function of three variables by placing 0, 1, C, or C' on each data input.

7) Cascading and expansion

Build larger MUXes from smaller ones:

  • Tree of 2:1 MUXes: three 2:1 stages form an 8:1 MUX
  • Dual 4:1 + 2:1: two 74153 chips plus one 2:1 stage → 8:1
  • Active-low enable: cascade with enable pins to build hierarchical selectors

MUX-based design trades extra select wiring for regular structure — useful in FPGAs and custom ICs where MUX primitives are abundant.

The inverse device — routing one input to many outputs — is the demultiplexer.

8) Applications

  • Data routing: bus arbitration, register file read ports
  • Parallel-to-serial: serialize multiple bit lines through one wire
  • Logic synthesis: LUTs in FPGAs are small MUX-based lookup tables
  • ALU slices: select arithmetic or logic operation results
  • Function implementation: realise any minterm cover with one MUX plus wiring
