1) Algebraic and number-theoretic foundations

Modern cryptography rests on discrete mathematics: finite fields \(\mathbb{F}_p\), \(\mathbb{F}_{p^n}\), groups, and hardness assumptions. A one-way function is easy to compute but hard to invert; a trapdoor function is invertible only with secret information.

Modular arithmetic: In \(\mathbb{Z}/n\mathbb{Z}\), addition and multiplication are well-defined. Euler's theorem: if \(\gcd(a,n) = 1\), then \(a^{\varphi(n)} \equiv 1 \pmod{n}\).
Discrete logarithm problem (DLP): Given generator \(g\) and \(h = g^x\) in a group \(G\), finding \(x\) is believed hard for large \(|G|\). Foundation of Diffie–Hellman and ElGamal.

Prerequisite: Group Theory, Elementary Algebra, and Combinatorics.

2) Symmetric cryptography

Symmetric-key schemes use the same secret key for encryption and decryption. Security depends on key length, algorithm design, and resistance to cryptanalysis.

Primitive Role Example
Block cipher Encrypt fixed-size blocks AES-128/256 (128-bit blocks)
Stream cipher XOR keystream with plaintext ChaCha20
Hash function Fixed-length digest, one-way SHA-256, SHA-3
MAC Authenticated integrity HMAC-SHA256
AES structure: Substitution-permutation network with key schedule. Security margin against differential and linear cryptanalysis is analyzed via algebraic properties over \(\mathbb{F}_2\).
Convention: Never reuse a nonce with the same key in stream ciphers (e.g. ChaCha20-Poly1305). Nonce reuse leaks the XOR of plaintexts.

3) Public-key cryptography and RSA

Public-key cryptography uses a key pair: public key for encryption/verification, private key for decryption/signing. Security reduces to mathematical hardness assumptions.

RSA: Choose distinct primes \(p, q\); \(n = pq\), \(\varphi(n) = (p-1)(q-1)\). Pick \(e\) with \(\gcd(e, \varphi(n)) = 1\); find \(d \equiv e^{-1} \pmod{\varphi(n)}\).
Encrypt: \(c \equiv m^e \pmod{n}\). Decrypt: \(m \equiv c^d \pmod{n}\).
Security relies on factoring \(n\) (or computing \(\varphi(n)\)).
Padding (OAEP): Raw RSA is not semantically secure. OAEP (Optimal Asymmetric Encryption Padding) is provably secure in the random oracle model under RSA assumptions.

Key sizes

RSA-2048 is a common minimum; RSA-3072+ for long-term security. Symmetric AES-256 offers \(\sim 128\)-bit classical security equivalent.

Hybrid encryption

RSA/ECDH establishes a session key; bulk data encrypted with AES. TLS 1.3 uses ECDHE + AES-GCM or ChaCha20-Poly1305.

4) Diffie–Hellman and elliptic curve cryptography

Diffie–Hellman key exchange: Alice and Bob agree on prime \(p\) and generator \(g\). Alice sends \(g^a\), Bob sends \(g^b\); both compute shared secret \(g^{ab} \pmod{p}\). Eavesdropper faces DLP.

Elliptic curve group: Points on \(E : y^2 = x^3 + ax + b\) over \(\mathbb{F}_p\) form an abelian group under chord-and-tangent addition. Scalar multiplication \(kP\) is one-way if DLP is hard.
ECDH / ECDSA: Curve25519 (X25519) for key exchange; NIST P-256 or Ed25519 for signatures. Smaller keys than RSA at comparable security: 256-bit ECC \(\approx\) 3072-bit RSA.
Common mistake: Using non-standard curves or improper point validation opens invalid-curve attacks. Prefer well-vetted curves (Curve25519, P-256 with full validation).

5) Digital signatures and authentication

A digital signature provides authenticity and non-repudiation: only the holder of the private key can sign; anyone with the public key can verify.

ECDSA: Sign message hash \(h\): choose random \(k\); compute \((x_1, y_1) = kG\); \(r = x_1 \bmod n\); \(s = k^{-1}(h + rd) \bmod n\). Signature is \((r, s)\). Verification uses public key \(Q = dG\).
EdDSA (Ed25519): Deterministic nonce derivation eliminates \(k\) reuse vulnerabilities. Faster and simpler than ECDSA for many implementations.

Hash-and-sign paradigm

Sign \(H(m)\), not \(m\) directly. Collision resistance of \(H\) is essential. SHA-256 or SHA-3 are standard choices.

Zero-knowledge proofs

Prove knowledge of a secret without revealing it. zk-SNARKs and zk-STARKs enable privacy-preserving verification in blockchains and credentials.

6) Protocols, attacks, and provable security

Cryptographic protocols (TLS, SSH, Signal) compose primitives under precise threat models. Security proofs reduce protocol safety to primitive assumptions in idealized models.

Attack classes: Ciphertext-only, known-plaintext, chosen-plaintext (CPA), chosen-ciphertext (CCA). Semantic security (IND-CPA) requires indistinguishability of ciphertexts.
Side-channel attacks: Timing, power analysis, and cache attacks extract key material without breaking mathematics. Constant-time implementations are mandatory for production crypto.
Threat Mitigation
Man-in-the-middle Certificate chains, certificate pinning
Replay attacks Nonces, timestamps, sequence numbers
Weak randomness CSPRNG (e.g. /dev/urandom, RDRAND)
Quantum adversary Post-quantum algorithms (see below)

7) Post-quantum cryptography and research outlook

Shor's algorithm

Quantum computers break RSA, DH, and ECC by solving factoring and DLP in polynomial time. Symmetric crypto (AES, SHA) needs larger key sizes but survives.

Lattice-based crypto

CRYSTALS-Kyber (KEM) and CRYSTALS-Dilithium (signatures) are NIST-standardized. Security based on Learning With Errors (LWE) and module-LWE problems.

Homomorphic encryption

Compute on encrypted data. Fully homomorphic encryption (FHE) enables cloud processing without decryption — active research area with performance trade-offs.

Multi-party computation

Parties jointly compute a function on private inputs. Combines with Game Theory for incentive-compatible protocols.

Related: Group Theory, Algebraic Number Theory, Combinatorics.
