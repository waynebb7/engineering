1) About this topic (read first)

Random sequence extrapolation sounds like extending a list of random numbers into the future. In proper probability theory, true randomness has no predictable pattern — the next value does not depend deterministically on the past. Extrapolation succeeds only when the sequence is not actually random, or when you model uncertainty rather than point-predict the next term.

Red flag: Products or papers claiming to "extrapolate random sequences" often misuse the word random for pseudorandom, noisy but structured, or partially observed data.

2) Random vs pseudorandom

Random sequence

Outcomes from a stochastic process; future values are independent (or have only probabilistic dependence). No algorithm can guarantee correct next-value prediction beyond the distribution mean.

Pseudorandom sequence

Deterministic algorithm \(x_{n+1} = f(x_n)\) mimicking randomness (LCG, Mersenne Twister). Extrapolation is possible in principle if you discover the generator — bad for cryptography, fine for simulations.

See Number Systems and Cryptography for why predictability matters in keys.

3) What extrapolation means in statistics

Extrapolation extends a fitted model outside the range of observed inputs. For time series \(\{y_t\}\), models like ARIMA or state-space Kalman filters forecast distributions: \(\mathbb{E}[y_{T+1} \mid \text{past}]\) and confidence intervals — not guaranteed exact "next random number."

Key idea: Forecasting uses structure (trend, seasonality, autocorrelation). Pure white noise \(y_t\) i.i.d. has best forecast \(\mathbb{E}[y_{T+1}] = \mu\) with irreducible error equal to the variance.

4) White noise: nothing to extrapolate

If \(X_1, X_2, \ldots\) are independent with mean 0 and variance \(\sigma^2\), knowledge of \(X_1,\ldots,X_n\) does not improve prediction of \(X_{n+1}\) beyond 0. The sequence has no memory.

This is the mathematical sense in which random sequence extrapolation is impossible — there is no additional deterministic signal to extract.

5) When prediction looks like "random extrapolation"

  • AR(1) process: \(X_{t+1} = \phi X_t + \varepsilon_t\) — past helps if \(|\phi| > 0\), but noise \(\varepsilon_t\) remains random.
  • Random walk: \(S_{n+1} = S_n + Z_{n+1}\) — best forecast is \(S_n\) (last value), variance grows.
  • Compression: Detecting a repeating PRNG seed from output — cryptanalysis, not statistics.
  • ML on chaotic systems: Short-horizon prediction possible; long-horizon fails (sensitivity to initial conditions).

6) Regression to the mean (common trap)

After an unusually high random draw, the next value often looks "lower" — not because the sequence compensates, but because extremes are rare. Gamblers' fallacy claims independent events "balance out"; they do not.

Exam tip: Distinguish independent trials from autocorrelated processes. Extrapolation methods differ completely.

8) Quick knowledge check

  1. Why can't you deterministically extrapolate a truly i.i.d. random sequence?
  2. What is the difference between random and pseudorandom?
  3. For white noise with mean 5, what is the best forecast of the next value?
  4. What does an AR(1) model add that white noise lacks?
  5. Why is "extrapolation" a misleading word for OTP key bits?
Show answers
  1. In a truly i.i.d. sequence, each future value is independent of the past — there is no memory and no deterministic pattern. Knowledge of \(X_1, \ldots, X_n\) does not improve point prediction of \(X_{n+1}\) beyond the distribution mean; only probabilistic forecasts (mean, variance) are possible.
  2. A random sequence comes from a stochastic process — future values have only probabilistic dependence on the past. A pseudorandom sequence is generated by a deterministic algorithm (e.g. LCG, Mersenne Twister) that mimics randomness; if you discover the generator, extrapolation becomes possible in principle.
  3. The best forecast is the mean: \(\mathbb{E}[y_{T+1}] = \mu = 5\). White noise has no memory — past observations do not shift the point prediction. Forecast uncertainty (variance \(\sigma^2\)) remains irreducible.
  4. An AR(1) model \(X_{t+1} = \phi X_t + \varepsilon_t\) adds autocorrelation — past values help predict the next value when \(|\phi| > 0\). White noise lacks this memory; the AR(1) signal component is predictable even though the noise \(\varepsilon_t\) is not.
  5. OTP key bits must be truly random, uniform, and one-time — there is no pattern or structure to fit or extend. Calling key recovery \"extrapolation\" is misleading: predicting key bits from ciphertext would break perfect secrecy; the key is not a noisy trend to forecast.
