1) About this topic (read first)

Unlike other entries in the AI Only catalog section, the one-time pad (OTP) is real, rigorous cryptography. It was proved information-theoretically secure by Shannon (1949). It appears in this section likely because the catalog title emphasises "decryption" in a sensational way — but the mathematics is standard and taught in security courses.

Placement note: For digital logic context see Boolean Algebra and Cryptography (postgraduate page).

2) How the one-time pad works

Represent plaintext and key as bit strings of equal length \(n\). Encryption: \[ c_i = p_i \oplus k_i \quad \text{for } i = 1,\ldots,n \] where \(\oplus\) is XOR (addition mod 2). Decryption uses the same operation: \[ p_i = c_i \oplus k_i. \] XOR is self-inverse: \((p \oplus k) \oplus k = p\).

Example

\(p = 1011\), \(k = 0110\) → \(c = 1101\). Decrypt: \(1101 \oplus 0110 = 1011\).

Key requirement

Key must be random, as long as the message, and never reused.

3) Why decryption without the key is impossible

If \(k\) is uniform over \(\{0,1\}^n\), then for any fixed ciphertext \(c\), every plaintext \(p\) is equally likely (because \(k = c \oplus p\) is equally likely). An attacker learns zero Shannon information about \(p\) from \(c\) alone.

Not computational hardness: OTP security does not rely on "hard maths problems" like factoring. Even an infinitely powerful computer cannot break OTP without the key — if the pad rules are followed.

4) Shannon's perfect secrecy definition

A cipher has perfect secrecy if for all plaintexts \(p_1, p_2\) and all ciphertexts \(c\): \[ \Pr[P = p_1 \mid C = c] = \Pr[P = p_2 \mid C = c]. \] OTP with a uniform one-time key achieves this. No other cipher can be perfectly secure with a shorter key than the message.

5) The two-time pad catastrophe (key reuse)

If the same key encrypts two messages \(p_1\) and \(p_2\): \[ c_1 \oplus c_2 = p_1 \oplus p_2. \] The key cancels. Partial knowledge of one message (language redundancy) often reveals the other — this broke Soviet OTP traffic in the VENONA project.

Lesson: "One-time" is not optional. Reuse turns perfect security into a solvable puzzle.

6) Practical limitations (why OTP is rare online)

  • Key distribution: You need \(n\) random key bits per \(n\) message bits — securely delivered in advance.
  • Synchronisation: Sender and receiver must agree on key index; loss of alignment destroys decryption.
  • No authentication: OTP does not stop tampering; combine with MACs in real systems.
  • True randomness: Pseudorandom keys reduce security to computational assumptions — no longer perfect secrecy.

Modern TLS uses computational cryptography (AES, ECDH) because OTP key logistics are impractical at internet scale.
