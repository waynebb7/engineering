# Contributor map

Quick reference for navigating and extending the Engineering Knowledge Hub repository.

## Start here

| If you want to… | Open |
|-----------------|------|
| Browse the site locally | `index.html` → [http://localhost:8080](http://localhost:8080) after `python -m http.server 8080` |
| Add a calculator | `calculators/{category}/` + `assets/js/calculator-registry.js` |
| Add a lesson / topic | `learn/{subject}/{level}/` + subject `index.html` catalog |
| Edit reference tables | `reference/electrical/`, `reference/physics/`, etc. |
| Add a PDF to the library | `reference/documents/files/` + `manifest.json` |
| Change site chrome / nav | `assets/js/site-layout.js`, `assets/css/corporate.css` |
| Regenerate catalogs or maps | `scripts/` (see commands below) |

Full layout details: [PROJECT-STRUCTURE.md](../PROJECT-STRUCTURE.md)

## Folder map

```
engineering/
├── index.html              ← Site hub (users start here)
├── progress.html           ← Learning progress dashboard
├── feedback.html           ← Feedback form
│
├── assets/                 ← Shared CSS and JS (all pages link here)
├── calculators/            ← Calculator pages by category
├── learn/                  ← Teaching pages (math, physics, quantum)
├── reference/              ← Variables, equations, document library
├── maps/                   ← Prerequisite graphs + topic JSON
├── legacy/                 ← Older drill-down explorers
├── meta/                   ← Generated subject tree
├── scripts/                ← Build and maintenance scripts
└── docs/                   ← Documentation and chat exports
```

## What to ignore in the file tree

Hundreds of **redirect stub** HTML files remain at old paths (e.g. `dc_power_calculator.html`, `quantum/…`, `basic_physics/…`) so bookmarks and old GitHub Pages URLs keep working. They meta-refresh to the new location under `calculators/` or `learn/`.

**Real content** lives in the folders above — not in the flat root HTML list.

Cursor/VS Code hides these stubs automatically via `.vscode/settings.json` (regenerated by `scripts/build-vscode-excludes.py`).

## Common maintenance commands

After editing catalogs, maps, or redirect stubs:

```bash
python scripts/build-topic-catalog.py
python scripts/build-topic-progression.py
python scripts/build-prereq-node-pages.py
python scripts/build-prereq-maps.py
python scripts/build-drilldown-explorers.py
python scripts/apply-catalog-progression.py
python scripts/ensure-redirect-stubs.py
python scripts/check-links.py --scope core
```

`ensure-redirect-stubs.py` also refreshes IDE hide rules for redirect stubs.

## Adding content (checklist)

### New calculator

1. Create HTML shell under `calculators/{category}/` with `class="calculator-page"` and `data-calc="…"`.
2. Register in `assets/js/calculator-registry.js`.
3. Link from `calculators/index.html` and/or `index.html` hub card.
4. Run `python scripts/check-links.py --scope core`.

### New lesson page

1. Create HTML under `learn/{subject}/{level}/` with `class="content-page"`.
2. Add a catalog link in `learn/{subject}/index.html`.
3. Regenerate catalog/progression scripts (see above).
4. Run link and catalog verification scripts.

### New document (PDF)

1. Copy file to `reference/documents/files/`.
2. Register in `reference/documents/manifest.json`.
3. Run `python scripts/build-documents-manifest.py`.

## Quality checks

```bash
python scripts/check-links.py --scope core
python scripts/verify-catalog-links.py
python scripts/verify-redirect-stubs.py
node --check assets/js/site-layout.js
```

CI runs these on every push — see `.github/workflows/ci.yml`.
