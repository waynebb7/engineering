Gray Code

Digital logic guide. Reflected binary code where only one bit changes between consecutive values — essential for rotary encoders and glitch-free counting.

← Pure Math Subjects Catalog

1) What is Gray code?

Gray code (reflected binary code) is a binary numbering system in which two consecutive values differ in exactly one bit. Standard binary (natural binary) does not have this property: for example, 0111 → 1000 changes four bits at once.

Key idea: Gray code is not weighted like ordinary binary — you cannot read a Gray codeword as a sum of place values. It is an ordering of binary strings optimised for smooth transitions.

Gray code connects to Number Systems and Encoders used in position sensing.

2) Why only one bit must change

In real hardware, signals do not switch instantaneously. When multiple bits change simultaneously, brief intermediate states (glitches) can appear before the outputs settle.

  • Rotary encoders: As a shaft rotates, sensors read successive positions — single-bit change prevents false readings during transition.
  • Asynchronous sampling: If a counter is read while updating, Gray code limits the error to at most one bit.
  • Karnaugh maps: Adjacent cells use Gray ordering so variables change one at a time — see Karnaugh Mapping.
Example glitch: Binary 3 → 4 is 011 → 100. A decoder might momentarily see 010, 001, or other spurious values. Gray 010 → 110 changes only the MSB.

3) Building reflected binary code

The classic construction is reflect and prefix. Start with 0, 1, then reflect the list, prefix the top half with 0 and the bottom half with 1.

3-bit construction:
Start: 0, 1
Reflect: 0, 1, 1, 0
Prefix: 000, 001, 011, 010, 110, 111, 101, 100 (decimal 0–7)

Each reflection doubles the length. After n steps you have n-bit Gray codes for 2ⁿ values. The sequence is cyclic: the last codeword also differs from the first by one bit.

4) Gray code to binary conversion

To decode a Gray codeword Gn-1…G0 to binary Bn-1…B0:

  1. MSB is unchanged: Bn-1 = Gn-1
  2. Each subsequent bit is XOR of the Gray bit and the previous binary bit: Bi = Gi ⊕ Bi+1
Worked example: Gray 1011 → Binary
B₃ = G₃ = 1
B₂ = G₂ ⊕ B₃ = 0 ⊕ 1 = 1
B₁ = G₁ ⊕ B₂ = 1 ⊕ 1 = 0
B₀ = G₀ ⊕ B₁ = 1 ⊕ 0 = 1
Result: binary 1101 = decimal 13

5) Binary to Gray conversion

To encode binary Bn-1…B0 as Gray Gn-1…G0:

  1. MSB is unchanged: Gn-1 = Bn-1
  2. Each Gray bit is XOR of adjacent binary bits: Gi = Bi ⊕ Bi+1
Worked example: Binary 1101 → Gray
G₃ = B₃ = 1
G₂ = B₂ ⊕ B₃ = 1 ⊕ 1 = 0
G₁ = B₁ ⊕ B₂ = 0 ⊕ 1 = 1
G₀ = B₀ ⊕ B₁ = 1 ⊕ 0 = 1
Result: Gray 1011

Compact formula for the least significant bit: G₀ = B₀ ⊕ B₁; for the most significant: Gn-1 = Bn-1.

7) Rotary encoders

A rotary encoder is a sensor on a spinning shaft that outputs a digital position code. Absolute encoders use Gray-coded tracks so that as the disk rotates from one sector to the next, only one photodiode or contact changes state.

How it works

  1. Concentric tracks encode each bit of position
  2. Tracks are patterned in Gray order around the disk
  3. Optical or magnetic sensors read all tracks simultaneously
  4. Decoder converts Gray output to binary angle or step count

Encoder types

  • Absolute: unique Gray code per angular sector
  • Incremental: A/B quadrature signals (related idea: one transition at a time)
Engineering benefit: Without Gray coding, a position near a binary rollover could report a wildly wrong angle for one sample — dangerous in robotics, CNC machines, and aerospace actuators.
