Encoders

Digital logic guide. Binary and priority encoders, active-low inputs, and converting one-hot lines into compact binary codes.

← Pure Math Subjects Catalog

1) What is an encoder?

An encoder converts a one-hot (or one-active) input pattern into a compact binary output code. If exactly one of \(2^n\) input lines is active, an \(n\)-bit encoder produces the binary index of that line.

  • Input: \(2^n\) lines, typically one asserted at a time
  • Output: an \(n\)-bit binary word identifying the active input
  • Inverse of a decoder: decoders expand binary codes to one-hot outputs; encoders compress one-hot to binary
Example: An 8-to-3 encoder has inputs \(I_0 \ldots I_7\). If \(I_5 = 1\) and all others are 0, outputs are \(Y_2 Y_1 Y_0 = 101_2\) (decimal 5).

2) Binary encoder (8-to-3)

A plain binary encoder assumes exactly one input is logic 1. Each output bit is the OR of selected inputs:

\(I_0\)000
\(I_1\)001
\(I_2\)010
\(I_3\)011
\(I_4\)100
\(I_5\)101
\(I_6\)110
\(I_7\)111

Boolean equations for an 8-to-3 encoder:

  • \(Y_0 = I_1 + I_3 + I_5 + I_7\)
  • \(Y_1 = I_2 + I_3 + I_6 + I_7\)
  • \(Y_2 = I_4 + I_5 + I_6 + I_7\)

Prerequisite: Logic Gates, Boolean Algebra.

3) Priority encoder

When more than one input may be active, a plain encoder produces an ambiguous or meaningless code. A priority encoder always reports the index of the highest-priority active input (conventionally the input with the largest index).

A typical 8-to-3 priority encoder also provides:

  • Valid output (\(V\)): asserted when at least one input is active
  • Priority rule: if \(I_5\) and \(I_2\) are both 1, output is 101 (5 wins over 2)
Why it matters: Keyboard matrices, interrupt controllers, and bus arbitration all need priority encoding when multiple requests can arrive simultaneously.
Common mistake: Using a plain encoder on a keyboard or interrupt bus without priority logic — multiple keys pressed at once will corrupt the output code.

4) Active-low inputs

Many IC encoders use active-low inputs: a line is considered active when it is driven to logic 0, not logic 1. This matches open-collector or wired-OR interrupt lines common in microcontrollers.

Active-high view

Input \(I_k = 1\) means "line \(k\) is selected."

Active-low view

Input \(\overline{I_k} = 0\) means "line \(k\) is selected." Internally, invert before OR logic.

For active-low inputs, each OR gate in the encoder equation uses inverted inputs: \(Y_0 = \overline{I_1} \cdot \overline{I_3} \cdot \ldots\) is wrong — instead, active-low means "selected when low," so the internal logic treats \(\overline{I_k} = 0\) as the asserted condition. Datasheets label these pins with overbars (e.g. \(\overline{I_7}\)).

Reading datasheets: Check whether outputs are also active-low. A 74148 priority encoder uses active-low inputs and active-low outputs; you may need inverters at the interface.

5) Worked example: keyboard row scan

A 4×4 keyboard matrix produces one-hot row signals when a single key is pressed. Connect rows to a 4-to-2 priority encoder to obtain a 2-bit row address:

  1. Only one row line goes low (active-low) when one key in that row is pressed
  2. Priority encoder outputs \(R_1 R_0\) identifying the row (0–3)
  3. Column scan (via a multiplexer) identifies the column
  4. Combine row and column codes to look up the key in a ROM or lookup table
Worked result: Row 2 is active (third row, zero-indexed). Priority encoder outputs 10. If column scan returns 01, the key at row 2, column 1 is identified.

6) Encoder vs multiplexer vs decoder

Encoder

Many inputs → few outputs. Compresses one-hot to binary index.

Decoder

Few inputs → many outputs. Expands binary index to one-hot. See Decoders.

  • Multiplexer: selects one of many data inputs; address lines choose the source
  • Encoder: reports which single line is active; no data path, only address generation
  • Cascading: large priority encoders (16-to-4, 32-to-5) cascade smaller blocks with group-enable signals

7) Design notes and limitations

  • All-zero input: plain encoders have no valid output when no input is active; priority encoders add a \(V\) flag
  • Glitch-free output: use registered outputs if the encoder feeds synchronous logic
  • Propagation delay: priority chains add delay proportional to input count; compare with tree structures for high-speed designs
  • Don't-care conditions: if only one input can ever be active, plain encoders suffice and are simpler
Timing hazard: When two inputs change nearly simultaneously, a priority encoder may briefly output an intermediate code before settling. Debounce or register outputs in safety-critical systems.
