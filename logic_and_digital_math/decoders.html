Decoders

Digital logic guide. n-to-2n decoders, enable inputs, and using decoders as general combinational logic building blocks.

← Pure Math Subjects Catalog

1) What is a decoder?

A decoder converts an \(n\)-bit binary input address into \(2^n\) one-hot output lines. Exactly one output is active (logic 1) for each valid input combination; all others are 0.

  • 2-to-4 decoder: 2 address inputs → 4 outputs (\(Y_0 \ldots Y_3\))
  • 3-to-8 decoder: 3 address inputs → 8 outputs
  • General form: \(n\) inputs, \(2^n\) outputs — an n-to-2n decoder
Core idea: Output \(Y_k\) is the minterm \(m_k\) of the address variables. A decoder literally generates all minterms of its inputs.

2) Truth table: 2-to-4 decoder

\(A_1\)\(A_0\)\(Y_0\)\(Y_1\)\(Y_2\)\(Y_3\)
001000
010100
100010
110001

Boolean equations (each output is one minterm):

  • \(Y_0 = \overline{A_1}\,\overline{A_0}\)
  • \(Y_1 = \overline{A_1}\,A_0\)
  • \(Y_2 = A_1\,\overline{A_0}\)
  • \(Y_3 = A_1\,A_0\)

Prerequisite: Truth Tables, Logic Gates.

3) Enable input

Practical decoder ICs include an enable input (sometimes labelled \(\overline{E}\), G, or STROBE). When enable is inactive, all outputs are forced to 0 regardless of the address — useful for bus sharing and chip selection.

Active-high enable

\(EN = 1\): decoder operates normally. \(EN = 0\): all outputs off.

Active-low enable

\(\overline{E} = 0\): decoder active. \(\overline{E} = 1\): all outputs off. Common in 74xx series parts.

Modified equation for output \(Y_k\) with active-high enable:

\(Y_k = EN \cdot m_k(A_{n-1}, \ldots, A_0)\)

Memory addressing: In a simple ROM, address bits feed a decoder; the selected output line enables one word of stored data. Enable gates the entire decoder off during inactive bus cycles.

4) Building logic with decoders

Because each decoder output is a minterm, you can implement any combinational function by OR-ing the appropriate decoder outputs. This is a systematic alternative to K-map minimisation.

Procedure:

  1. Express the function as a sum of minterms (SOP) from its truth table
  2. Connect address inputs to the decoder
  3. OR together the decoder output lines corresponding to minterms where \(f = 1\)
Example: \(f(A,B,C) = \Sigma(1, 3, 5, 7)\) using a 3-to-8 decoder:
Connect \(A, B, C\) to address inputs. OR outputs \(Y_1, Y_3, Y_5, Y_7\). Result: \(f = C\) (all minterms where \(C=1\)).

A decoder plus OR gates implements any function of \(n\) variables — the same expressive power as a multiplexer-based implementation, but often with more wiring for sparse functions.

5) Worked example: 7-segment driver

Display the digits 0–9 using a BCD-to-7-segment decoder (a specialised decoder with 4 inputs and 7 segment outputs):

  1. 4-bit BCD input \(D_3 D_2 D_1 D_0\) represents digits 0–9
  2. Each segment output is 1 (or 0, depending on common-anode/cathode) for the segments lit in that digit
  3. Internally, each segment equation is a sum of minterms — exactly the decoder-plus-OR structure
Design insight: A 4-to-16 decoder plus seven OR gates can implement any four-variable segment pattern. Dedicated BCD-to-7-segment ICs (e.g. 7447) embed the optimised minterm sums in one package.

6) Cascading decoders

Build larger decoders from smaller ones using enable lines as a hierarchy:

  • 4-to-16 from two 3-to-8 decoders: use one address bit to select which chip is enabled; remaining three bits pick the output within that chip
  • 5-to-32: one 2-to-4 decoder generates four enable signals; four 3-to-8 decoders each drive eight outputs

Advantage

Uses standard ICs; modular and easy to breadboard.

Trade-off

Extra propagation delay through the enable tree. See Circuit Complexity.

7) Decoder vs demultiplexer

A demultiplexer (demux) routes one data input to one of many outputs based on an address. A decoder is equivalent to a demux with the data input tied to logic 1 (and enable active):

  • Decoder: generates minterms — outputs are always 0 or 1 based on address alone
  • Demux: passes an external data bit to the selected output; others receive 0
  • See Demultiplexers for the data-routing view
Common mistake: Forgetting the enable pin on cascaded decoders — if the upper decoder's enable is stuck inactive, half the outputs never activate.
